## 2026-05-11 ‚Äî M9.1.a: tool_sources config + manifest.json schema + ToolSyncScheduler (new core/pkg/toolregistry)

**PR**: pending ‚Äî to be opened in Phase 5b
**Merged**: pending ‚Äî to be merged in Phase 7a

### Context

M9.1.a opens the M9 milestone family. The roadmap text on M9.1 is too
large to ship atomically (operator config schema + manifest decoding +
clone/pull I/O + scheduler + effective-toolset recompute + runtime
hot-reload signal with grace period ‚Äî six orthogonal surfaces), so the
operator chose a 2-way decomposition along the data/runtime seam
(commit `8bf1018`):

- **M9.1.a** ‚Äî the data + sync layer: operator `tool_sources` config,
  per-tool `manifest.json` schema, and `ToolSyncScheduler` that
  clones / pulls each source per its pull policy and emits
  `toolregistry.source_synced` / `toolregistry.source_failed` events.
- **M9.1.b** ‚Äî the runtime layer: effective-toolset recompute on each
  `source_synced` event + a hot-reload signal to running runtimes with
  configurable grace period (in-flight `InvokeTool` calls complete on
  the old version while new invocations resolve against the new
  toolset).

The new package is `core/pkg/toolregistry/`. It is intentionally
self-contained ‚Äî no imports from `core/pkg/config`, no audit emission,
no runtime coupling. Downstream subscribers (M9.1.b, M9.2 resolver,
M9.7 audit) join via `eventbus`.

Files:

- `doc.go`, `errors.go` ‚Äî package documentation + sentinel errors.
- `manifest.go` ‚Äî `Manifest` struct + `DecodeManifest` (strict JSON
  decode, `DisallowUnknownFields`, required-field validation,
  defensive deep-copy of reference-typed fields) +
  `LoadManifestFromFile` (file read + decode + source-stamping with
  collision detection).
- `config.go` ‚Äî `SourceConfig` + closed-set `SourceKind`
  (`git` / `local` / `hosted`) and `PullPolicy`
  (`on-boot` / `cron` / `on-demand`) + per-source `Validate()` +
  `ValidateSources()` (covers duplicate names) + `EffectiveBranch()`
  defaulting to `main` + `CloneSources()` defensive copy.
- `yaml.go` ‚Äî `DecodeSourcesYAML` (strict yaml.v3 `KnownFields(true)`,
  multi-document rejection) + `LoadSourcesYAMLFromFile`.
- `events.go` ‚Äî `TopicSourceSynced` / `TopicSourceFailed` constants +
  `SourceSynced` (sourceName, syncedAt, localPath, correlationID) and
  `SourceFailed` (sourceName, failedAt, errorType, phase,
  correlationID) payloads.
- `scheduler.go` ‚Äî `Scheduler` orchestrator with five seams (`FS`,
  `GitClient`, `Clock`, `SignatureVerifier`, `Publisher`) +
  `AuthSecretResolver` per-call resolver + optional `Logger`. `New`
  panics on nil required deps; `SyncOnce(ctx, sourceName)` performs
  one clone-or-pull cycle and emits the appropriate event.
- `osfs.go` ‚Äî production `OSFS` adapter forwarding to `os.*`.

Tests cover: every manifest decode failure path (malformed JSON,
unknown field, missing-required for each of `name` / `version` /
`capabilities` / `schema`, trailing JSON garbage including raw-junk
trailing bytes), every config failure path (invalid name including
path-traversal payloads, kind / pull policy / cron spec, git
missing URL, local/hosted with URL or auth_secret, duplicate names),
every YAML failure path (unknown top-level key, unknown entry
field, multi-document, propagation of validation errors), every
sync phase failure (`path` / `auth` / `mkdir` / `stat` / `clone` /
`pull` / `signature`) with corresponding `SourceFailed.Phase` tag,
default `NoopSignatureVerifier` happy path, per-call
`AuthSecretResolver` invocation count, empty-resolved-auth
fail-fast, local-kind-missing-dir refusal, defensive deep-copy of
sources post-construction, defense-in-depth path-containment
assertion in `SyncOnce`, 16-goroutine concurrency on different
sources (parallel) and on the same source (serialised via
per-source mutex), source-grep AC (production files never import
`keeperslog.` or call `.Append(`), PII canary harness across every
failure phase, URL-credential leakage canary on `SourceFailed`
payload, leaf-error-type extraction over wrapped chains.

### Review iter-1 fixes

Codex + critic (parallel) flagged 7 Major + 8 Minor; the operator
chose "All findings" and the union was applied on the same branch
before commit:

- **M1 ‚Äî DecodeManifest accepts raw-junk trailing** ‚Äî `manifest.go`
  guard changed from `err == nil` to `!errors.Is(err, io.EOF)` so
  `{...} garbage` / `{...} [` are now refused as `ErrManifestParse`.
- **M2 ‚Äî local/hosted MkdirAll-before-Stat false positive** ‚Äî
  `scheduler.go` no longer calls `MkdirAll` for local / hosted
  kinds. A missing directory raises new sentinel
  `ErrLocalSourceMissing` with phase `stat`; population is the
  responsibility of M9.5 (local) and M9.4 (hosted).
- **M3 ‚Äî empty resolved auth silently demotes to anonymous** ‚Äî
  `resolveAuth` now wraps a resolver-returned `("", nil)` as
  `ErrEmptyResolvedAuth` (chained through `ErrAuthResolution`).
- **M4 ‚Äî path-traversal via source name** ‚Äî `SourceConfig.Validate`
  now enforces `^[a-zA-Z0-9_-]+$` (rejects `..`, `/`, `\`, `:`,
  `.`); `SyncOnce.safeLocalPath` adds defense-in-depth `filepath.Rel`
  containment check raising `ErrUnsafeLocalPath` if anything slips
  through. Today the config is operator-controlled; the validator
  is in place for M9.4 / M9.5 when AI-authored / lead-clicked
  names will land in this resolution path.
- **M5 ‚Äî publisher error on success silently dropped** ‚Äî asymmetric
  fix: `emitSuccess` now RETURNS the publisher error so SyncOnce
  surfaces it to the caller (M9.1.b's hot-reload signal must not
  be silently lost); `emitFailure` still SUPPRESSES the publisher
  error because the caller already has the original sync error in
  hand.
- **M6 ‚Äî non-existent `WithJiraBasicAuthResolver` referenced** ‚Äî
  doc-comment in `config.go` and this lesson entry updated to the
  actual `jira.BasicAuthSource` / `jira.WithBasicAuth` shape.
- **M7 ‚Äî signature-verifier holds per-source mutex during slow
  work** ‚Äî deliberately kept; documented on `Scheduler.SyncOnce`
  godoc as a trade-off (release-during-verify would let a
  concurrent pull race the verified tree). Revisit in M9.3 when
  cosign / minisign actually exposes the latency.
- **m1 ‚Äî PullPolicy enum bakes mutual exclusivity** ‚Äî added
  orthogonal `BootstrapOnBoot bool` flag to `SourceConfig` so
  "boot + cron" is expressible without splitting the policy enum
  or shipping a fourth value.
- **m2 ‚Äî resolveAuth before kind dispatch** ‚Äî `Validate` now
  rejects `auth_secret` on local / hosted via new sentinel
  `ErrAuthSecretNotAllowed`. The resolver path is git-only by
  construction.
- **m3 ‚Äî DataDir whitespace-only passes** ‚Äî `New` uses
  `strings.TrimSpace` consistent with the rest of the package.
- **m5 ‚Äî `SourceFailed.ErrorType` captures wrapper type** ‚Äî new
  `leafErrType(err)` helper walks `errors.Unwrap` to the deepest
  non-nil error so subscribers see `*url.Error` instead of
  `*fmt.wrapErrors`.
- **m6 ‚Äî `err == io.EOF` instead of `errors.Is`** ‚Äî yaml.go now
  consistently uses `errors.Is(err, io.EOF)`.
- **m7 ‚Äî `LoadManifestFromFile` returns ErrManifestParse for nil
  fs** ‚Äî switched to panic with named-arg message, matching the
  `New` discipline on nil deps.
- **m8 ‚Äî unused `dirEnt` / `ReadDir`** ‚Äî removed `ReadDir` from
  `FS` interface, `OSFS`, and `fakeFS`. The method was unused by
  M9.1.a (manifest decoding uses `ReadFile`); M9.1.b will add it
  back when it actually scans manifest directories.

### Pattern

1. **Atomize a multi-surface roadmap leaf along its natural seam BEFORE
   coding** ‚Äî M9.1 in the original ROADMAP text bundles six orthogonal
   surfaces. Mirror the M8.2 / M7.3 decomposition pattern: a separate
   `docs(roadmap):` commit on `main` that splits the leaf into named
   sub-items (`M9.1.a` / `M9.1.b`), then ship one sub-item per
   `ship-roadmap-item` invocation. The decomposition commit
   (`docs/ROADMAP-phase1.md:500-502`) is doc-only, lands directly on
   `main` (no PR ‚Äî the gated work is the per-sub-item PRs), and
   includes a body paragraph explaining the seam choice + why the
   single-PR alternative was rejected. The atomic-ship rule has a
   user-override channel via `AskUserQuestion` when an axis is
   genuinely indivisible.

2. **Seam-driven design: every side-effecting primitive is an
   interface, every default is no-op** ‚Äî the `Scheduler` consumes five
   interfaces (`FS`, `GitClient`, `Clock`, `SignatureVerifier`,
   `Publisher`) plus a callable `AuthSecretResolver`
   (`core/pkg/toolregistry/scheduler.go:21-118`). Each interface is
   defined in this package (not imported from elsewhere), uses the
   minimal subset of methods the scheduler actually calls, and has a
   hand-rolled fake in `fakes_test.go`. The default
   `SignatureVerifier` is the no-op `NoopSignatureVerifier{}` (zero
   value usable, returns nil for every input) ‚Äî this lets M9.1.a land
   the verification HOOK without committing to a specific signing
   scheme (cosign / minisign is M9.3's call). The production wiring
   for `FS` is a 4-method shim (`osfs.go`) ‚Äî adding a sixth seam later
   is mechanical.

3. **Per-call resolver beats process-global static for tenant-scoped
   values** ‚Äî `AuthSecretResolver` is `func(ctx, sourceName, ref)
(string, error)`, invoked on every `SyncOnce`
   (`scheduler.go` resolveAuth). The test
   `TestSyncOnce_AuthResolverPerCall` asserts the call count matches
   the number of SyncOnce invocations (not cached). Same lesson as
   M8.2.a's `jira.BasicAuthSource` interface (consumed via
   `jira.WithBasicAuth`): a per-tenant rotation takes effect on the
   next sync without restarting the process, and the scheduler never
   holds the plaintext secret across calls. The toolregistry side
   chose a function shape (vs an interface) because the resolver
   surface is one-method and stays simpler as a `func` type alias.

4. **PII discipline at the event-payload boundary, NOT at the logger**
   ‚Äî `SourceFailed` carries `ErrorType` (the Go type name via
   `fmt.Sprintf("%T", err)`) and `Phase` but NEVER the underlying
   error MESSAGE
   (`core/pkg/toolregistry/events.go:46-76`,
   `scheduler.go:374-395`). A failing git URL with embedded
   credentials would surface its message via `errors.Unwrap` if the
   payload carried `.Error()`; trimming the payload at the type-name
   boundary is what makes a subscriber that logs the event verbatim
   safe. The PII canary harness
   (`scheduler_test.go:TestPIIRedactionCanary_AcrossEveryFailurePath`)
   injects a synthetic `CANARY_PII_AUTH_TOKEN_‚Ä¶` into every failure
   phase (`mkdir` / `stat` / `clone` / `pull` / `signature` / `auth`)
   and asserts neither the event nor any logged kv pair contains the
   substring.

5. **Defensive deep-copy of reference-typed inputs in BOTH constructor
   AND on dispatch** ‚Äî `New` calls `CloneSources(sources)`
   (`config.go:148-156`) so caller-side mutation post-construction
   cannot bleed into the scheduler's state; `DecodeManifest` copies
   `Capabilities` (string slice) and `Schema` (byte slice) via the
   local `cloneStringSlice` / `cloneBytes` helpers
   (`manifest.go:104-118`) so the caller's `raw` buffer is no longer
   aliased. Mirrors the M7.1.c.c `cloneBotProfile` pattern. The test
   `TestDecodeManifest_DefensiveCopy` mutates the input buffer
   post-decode and asserts the returned manifest is unaffected.

6. **Per-source mutex serialises concurrent SyncOnce on the SAME
   source; different sources run in parallel** ‚Äî `Scheduler.perSourceMu`
   is a `map[string]*sync.Mutex` populated at construction
   (`scheduler.go:213-219`). `SyncOnce` takes the per-source mutex
   before touching the FS / git
   (`scheduler.go:308-310`). The concurrency tests
   (`TestSyncOnce_ConcurrencyDifferentSources`,
   `TestSyncOnce_ConcurrencySameSourceSerialised`) dispatch 16
   goroutines: the first asserts all 16 calls complete in parallel
   (all 16 sources clone exactly once); the second asserts the
   max-inflight count for the same source never exceeds 1 even
   under a 1ms artificial hold. The map is read-only after `New`, so
   no top-level lock is needed.

7. **Closed-set discriminators with named constants + per-discriminator
   cross-field validation** ‚Äî `SourceKind` and `PullPolicy` are
   typed string aliases with named constants
   (`config.go:13-58`). `SourceConfig.Validate` switches on the kind
   and policy enums, requiring `URL` for `git` (and forbidding it for
   `local` / `hosted`) and `CronSpec` for `cron` (and ignoring it for
   `on-boot` / `on-demand`)
   (`config.go:96-130`). A `default:` branch on each switch surfaces
   `ErrInvalidSourceKind` / `ErrInvalidPullPolicy` rather than a
   silent acceptance ‚Äî same loud-failure discipline as
   `runtime.AuthorityFromString`.

8. **Strict YAML decoding at the document root AND inside each entry**
   ‚Äî `DecodeSourcesYAML` uses yaml.v3's `KnownFields(true)` on the
   `sourcesYAMLDocument` wrapper, which propagates strictness to the
   nested `SourceConfig` entries (`yaml.go:36-44`). A typo in either
   the top-level key (`toolsources:` instead of `tool_sources:`) or
   an entry field (`pulled_policy:` instead of `pull_policy:`)
   surfaces an error rather than silently producing zero values. The
   multi-document guard (`yaml.go:51-58`) catches accidental
   merge-conflict artefacts.

9. **Sentinel-error sprawl is cheap when each sentinel names a
   concrete failure surface; callers `errors.Is` against the union**
   ‚Äî `errors.go` defines 16 sentinels covering source name / kind /
   policy / cron / URL allowance / data dir / unknown-source /
   manifest parse / manifest unknown field / manifest missing
   required / manifest source collision / signature verification /
   auth resolution / sync clone / sync pull / FS stat / FS mkdir.
   Each is a one-line `errors.New` with a doc-block explaining the
   condition that triggers it. `SyncOnce` wraps via
   `fmt.Errorf("%w: %w", ErrSyncClone, err)` so callers can
   `errors.Is(err, ErrSyncClone)` for kind-of-failure and
   `errors.Is(err, originalErr)` for the underlying cause. Mirrors
   the `core/pkg/eventbus` / `core/pkg/cron` sentinel discipline.

10. **Source-grep AC catches audit-emission drift early** ‚Äî the test
    `TestSourceGrepAC_NoAuditCallsInProductionSources`
    (`manifest_test.go`) reads every production file in the package
    and grep-asserts that `keeperslog.` and `.Append(` do not appear
    outside comments. Audit emission for the M9 family lives in M9.7;
    the data-layer package must remain audit-free so the scheduler is
    safe to import from M9.1.b's runtime layer without dragging in
    audit dependencies. A `containsOutsideComments` helper
    (`testhelpers_test.go`) skips both line-comment lines and
    in-line `//` tails so doc-blocks discussing the audit boundary
    don't trip the AC.

11. **The Stat-not-exist sentinel chain (`fs.ErrNotExist`) is the
    clone-vs-pull discriminant** ‚Äî
    `scheduler.go:347-358` calls `FS.Stat(<localPath>/.git)`; an
    `errors.Is(err, fs.ErrNotExist)` return triggers Clone, a nil
    error triggers Pull, and any other error aborts with `ErrFSStat`
    - `SourceFailed.Phase = "stat"`. The fake FS in `fakes_test.go`
      returns a `*fs.PathError{Err: fs.ErrNotExist}` for missing paths
      so the test matrix exercises the stdlib sentinel chain rather
      than a hand-rolled fake.

12. **Stop publish errors from masking sync success** ‚Äî
    `emitSuccess` / `emitFailure` swallow `Publisher.Publish` errors
    after logging them (`scheduler.go:351-371`). The SyncOnce caller
    already has the original sync outcome in hand; a downstream
    eventbus stall must not turn a successful clone into a reported
    failure. The test
    `TestSyncOnce_PublisherErrorDoesNotMaskSyncSuccess` asserts the
    SyncOnce return is nil even when the publisher returns
    `errSentinel`.
