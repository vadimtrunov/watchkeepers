## 2026-05-14 — M10.2: unified `wk` operator CLI subsuming `wk-tool`

**PR**: pending — to be opened in Phase 5b
**Merged**: pending — to be merged in Phase 7a

### Context

`docs/ROADMAP-phase1.md` §M10.2 asks for a single operator CLI covering
fifteen noun-groups (spawn / retire / list / inspect / logs /
tail-keepers-log / tool / tools sources / notebook / personality /
language / budget / approvals) plus a uniform `make wk CMD="…"` shortcut.
Following the M10.1 single-PR precedent, the operator chose the umbrella
shape over the natural a/b/c decomposition; the PR therefore lands the
full CLI contract in one go (`core/cmd/wk/`), absorbs the M9.5 / M9.6 /
M9.7 / M9.4.b operator surface from the retired `wk-tool` binary, and
pins the noun-group dispatcher + flag shapes for every command. Where a
shipped backend exists today (Keep HTTP surface for spawn / retire /
list / inspect / logs / tail / personality / language / budget show;
local-FS for notebook show / forget / export / import; `wk-tool`
migrations for tool local-install / rollback / hosted export / share)
the command is fully wired. Where the backend seam has not landed yet
(tool list / tool hosted list+show / tools sources \* / notebook
archive / notebook list-archives / budget set / approvals \*) the
command returns exit-3 with the
follow-up M-id named verbatim in the diagnostic — same pattern as
M10.1's stable Prometheus name-set pinned before every call site
landed.

`wk-tool` is removed in the same PR: `core/cmd/wk-tool/` is deleted and
the four migrated subcommands keep their flag shapes verbatim so
runbook snippets stay short — only the invocation prefix changes
(`wk-tool local-install …` → `wk tool local-install …`). The
`tools-local-install` make target preserves its name (M9.5 runbook
compatibility) and now forwards through `wk tool local-install` under
the hood.

### Pattern

1. **Hierarchical noun-group dispatcher beats flat subcommand list.**
   `core/cmd/wk/run.go` switches on the first arg (noun-group) and
   delegates to a per-noun-group dispatcher (`runTool`, `runNotebook`,
   `runPersonality`, …) which switches on the second arg (verb). The
   shape mirrors the ROADMAP text verbatim (`wk tool hosted export …`)
   and keeps each dispatcher small (≤60 LoC). A flat dispatcher (which
   M9.5's `wk-tool` used) does not scale past ~6 subcommands without
   either a generated parser or a third-party CLI lib.

2. **Centralise the Keep-client construction.** Every Keep-backed
   subcommand resolves the bearer token + base URL through one helper
   (`core/cmd/wk/keep.go` `withKeepClient(env, stderr, label, f)`).
   Missing env vars become exit-2 with a SINGLE diagnostic naming both
   `WATCHKEEPER_KEEP_BASE_URL` and `WATCHKEEPER_OPERATOR_TOKEN` (mirror
   the M9.5 accumulated-missing-flag pattern); the closure shape keeps
   each subcommand's error path one line. The token is intentionally
   ENV-only — `ps -ef` would leak it from argv.

3. **Exit code 3 = "contract present, backend pending"** (`run.go`
   `notWiredExit(stderr, label, followup)`). The CLI's contract surface
   is pinned NOW even where the seam is missing — every stub diagnostic
   names the follow-up M sub-item that will wire it (e.g. `M10.2.b` for
   `tools sources sync` / `notebook archive`, `M10.2.c` for `approvals
pending` / `budget set`). Operator runbooks can already reference the
   final CLI shape; the failure mode is loud and self-describing rather
   than silent or 0-stubbed. Mirror the M10.1 pattern (Prometheus name
   set defined up front; call sites land later).

4. **`make wk CMD="<noun-group> <verb> [flags]"` is the single
   composite shortcut.** Operators don't need a per-subcommand make
   target for every leaf — one wrapper that forwards CMD verbatim to
   `./bin/wk` covers every present and future verb. The Makefile keeps
   the legacy `tools-local-install` target name for runbook compatibility
   (M9.5 referenced it; M10.2 changes the underlying binary but not the
   target shape). A future `tools-rollback`, `tools-hosted-export`, etc.
   target are intentionally NOT in the Makefile — `make wk CMD="…"` is
   the supported general path; named targets only exist where a legacy
   runbook already references them.

5. **Read-merge-write for manifest field mutations**
   (`core/cmd/wk/run_manifest.go` `runManifestSet`). Personality and
   language `set` commands resolve <wk-id> → ManifestID → current
   ManifestVersion via two GETs, copy EVERY field of the in-memory
   ManifestVersion into a PutManifestVersionRequest with `version_no+1`
   and the target column overwritten, then PUT. The merge intentionally
   copies every field because the server has no PATCH API today; a
   missing field would let the jsonb column defaults overwrite a
   previously-set value. The closure shape (`apply
func(*ManifestVersion, string)`) keeps personality/language set
   one-line per field.

6. **Migrate-with-prefix-rename preserves all M9.5 tests** (`run_test.go`).
   The M9.5 / M9.6 / M9.4.b test bodies stayed verbatim under the new
   binary; only the subcommand args were prefixed with `"tool"` and one
   nested case with `"tool", "hosted", "export"`. Constants
   `dataDirEnvKey` / `localSourcesEnvKey` / `defaultLocalSource` /
   `errMissingFlags` / `jsonlPublisher` / `trustHintResolver` /
   `singleSourceLookup` / `hostedSourceLookup` / `shareSourceLookup`
   moved verbatim into `core/cmd/wk/run_tool.go`. The two redaction-harness
   tests (`TestRun_LocalInstall_JSONLStdoutOmitsFolderContentCanary`,
   `TestRun_Share_TokenCanaryNotEchoedToStdoutOrStderr`) carry forward
   exactly because they pin a runtime invariant that survives any
   prefix rename.

7. **`wk logs <wk>` is client-side filter, by design** (`run_logs.go`).
   Keep's `/v1/keepers-log` has no per-actor query parameter as of
   M10.2; the CLI requests up to 1000 events via `--limit` (server
   clamps) and drops rows whose `actor_watchkeeper_id != <wk>`. A
   future M10.2.d follow-up can add a server-side filter when keepers-log
   volume makes the 1000-row client filter inefficient. The default
   limit was sized to comfortably exceed a dev-workspace day of events
   while staying under Keep's documented `maxLogLimit` clamp.

8. **`wk tail-keepers-log` is the only long-lived subcommand**
   (`run_logs.go`). It uses `keepclient.Subscribe`'s SSE stream and
   prints each event frame's payload as one JSON line on stdout.
   Heartbeat frames are silently dropped by the keepclient scanner;
   the stream stays open until ctx is cancelled (SIGINT/SIGTERM,
   plumbed via `main.go`'s `signal.NotifyContext`). Both clean EOF
   (`io.EOF`) and a local close (`ErrStreamClosed`) terminate with
   exit 0 — only an unexpected error path emits exit 1.

9. **Notebook subcommands respect `$WATCHKEEPER_DATA`, not
   `$WATCHKEEPER_DATA_DIR`** (`run_notebook.go` package doc).
   `core/pkg/notebook/path.go`'s `agentDBPath` reads `WATCHKEEPER_DATA`
   verbatim from `os.Getenv`; the CLI does NOT shadow this (an attempt
   to push the toolregistry's `WATCHKEEPER_DATA_DIR` here would
   silently misroute the SQLite file). Documented honestly in the
   noun-group help and in this lesson so a future operator reading
   only the CLI help finds the rule without re-deriving it. The
   `notebook archive` + `list-archives` subcommands stub to exit-3
   today because they additionally need an `archivestore.Storer`
   configured via the Keep service's config — operator-side wiring
   for that storer lands in M10.2.b.

10. **`tools-local-install` make target is preserved through the
    rename** (`Makefile`). The wk-tool → wk migration changes the
    underlying binary; the Makefile target name stays so M9.5's
    `make tools-local-install FOLDER=… REASON=… OPERATOR=…
WATCHKEEPER_DATA_DIR=…` runbook snippet keeps working. The target
    body now calls `./bin/wk tool local-install` instead of
    `./bin/wk-tool local-install` and depends on `wk-build` instead
    of `wk-tool-build`. No operator-facing flag changes.

## 2026-05-14 — M10.1: Observability (`core/pkg/wklog` slog-JSON logger + `core/pkg/wkmetrics` Prometheus surface + Keep `/metrics` mount + outbox instrumentation)

**PR**: #144
**Merged**: 897ccc7 on 2026-05-14

### Context

`docs/ROADMAP-phase1.md` §M10.1 asks for a single observability surface
covering BOTH Prometheus metrics (token spend, latency histograms, tool
invocations, eventbus depth, rate-limit headroom) AND structured JSON
logs with correlation IDs and per-subsystem level config. The operator
explicitly chose a single-PR shape over the natural a/b/c
decomposition; the PR therefore lands two new `core/pkg/*` packages and
wires the Keep service's `/metrics` endpoint + outbox-publish counter
as the first real call sites. The remaining instrumentation hooks (LLM
token spend, tool invocations, Slack/Jira rate-limit headroom) are
defined as a stable Prometheus name/label contract in
`core/pkg/wkmetrics/wkmetrics.go` and wired into the respective
subsystems in their own follow-up sub-items as those packages land
their per-call seams.

### Review iter-1 fixes

- `codex review --base main --title …` (real OpenAI codex CLI; `omc`
  wrapper not installed, codex invoked directly). Output captured to

- **C1 (critic) — `statusRecorder` broke SSE streaming.** The original
  wrapper at `core/pkg/wkmetrics/wkmetrics.go` did not implement
  `http.Flusher` / `http.Hijacker`. Production wiring threads
  `/v1/subscribe` through `NewRouterWithMetrics`, which wraps every
  handler in `m.Instrument(...)`; the SSE handler at
  `core/internal/keep/server/handlers_subscribe.go:56` does
  `flusher, _ := w.(http.Flusher)` and short-circuits at line 64-68
  when the assertion fails. The bug was masked by the existing SSE
  tests (`handlers_subscribe_test.go:33` uses the no-metrics
  `NewRouter` constructor, so the wrap was never exercised in tests).
  Fix: `statusRecorder` now implements both `Flusher.Flush()` and
  `Hijacker.Hijack()`, each forwarding to the underlying writer when
  it supports the interface and degrading cleanly otherwise.
  Regression test `TestIter1_StatusRecorder_ProxiesFlusher` runs a
  flush through a real `httptest.NewServer` and asserts both that
  the wrapped writer IS a `http.Flusher` AND that `Flush()` reaches
  the underlying writer.

- **C2 (critic) / Codex-P1b — outbox success counter inflated on
  rollback.** The previous `tick()` at
  `core/internal/keep/publish/outbox.go` incremented
  `RecordOutboxPublished(success)` PER ROW after each stamp UPDATE,
  but before `tx.Commit`. Any subsequent stamp failure (or a Commit
  failure) triggered the deferred rollback, undoing every stamped row
  while the counter retained its bump — the next tick republished
  those rows and the counter double-counted. Fix: the loop now tracks
  `stampedSuccessfully` row count locally and only emits one
  `OutcomeSuccess` per stamped row AFTER `tx.Commit(ctx)` returns
  nil. On Commit failure the batch records ONE batch-level
  `OutcomeError` instead of per-row over-counting. The fix lives at
  `outbox.go:209-260`.

- **Codex-P1a — auth-failure traffic vanished from the HTTP
  histogram.** The original `NewRouterWithMetrics` mounted handlers
  as `authed(instrument(route, h))`, so 401 responses (missing /
  expired bearer) never contributed to
  `watchkeeper_http_request_duration_seconds`. Operators relying on
  the histogram for traffic + auth-failure visibility would see
  401-spam disappear from the panel. Fix: order swapped to
  `instrument(route, authed(h))` at `server.go:80-100` so 401s
  register at the metric layer.

- **M1 (critic) — `/metrics` outside auth leaks process info with
  no opt-out.** Default `ProcessCollector` + `GoCollector` registration
  in `wkmetrics.NewWithRegistry` made
  `process_resident_memory_bytes`, `process_open_fds`,
  `process_start_time_seconds`, `go_goroutines`, etc. publicly readable
  for any deployment that did not ACL `/metrics` at the network
  level. Fix: introduce `wkmetrics.Options.DisableProcessCollectors`
  and the `NewWithOptions(opts)` constructor (line 152-168). `New()`
  and `NewWithRegistry()` are thin wrappers that pass the
  zero-value Options (default collectors registered to preserve the
  out-of-the-box dashboard story). Operators behind a hostile
  network set `DisableProcessCollectors: true`. Regression test
  `TestIter1_NewWithOptions_DisableProcessCollectors` asserts (a)
  the process/Go families are absent when the flag is set AND
  (b) the owned `watchkeeper_*` families remain present.

- **M2 (critic) — `correlationHandler.WithGroup` nests correlation_id.**

### Pattern

1. **Nil-safe recorders are the single contract carrier for cross-cutting
   instrumentation** (`core/pkg/wkmetrics/wkmetrics.go:272-336`,
   methods `RecordTokenSpend`, `ObserveLLMRequest`,
   `RecordToolInvocation`, `SetEventBusDepth`, `SetRateLimitRemaining`,
   `ObserveHTTPRequest`, `RecordOutboxPublished`, `Instrument`). Every
   record-* method short-circuits on `m == nil`, so callers can hold a
   `*wkmetrics.Metrics` field and call it unconditionally. The diff at
the outbox call sites becomes one-line additions
(`core/internal/keep/publish/outbox.go:225, 251, 257`) with no
`if m != nil` noise. The contract is named explicitly in the
   package doc so a reviewer reading a future call site finds the rule
   without re-deriving it.

2. **The full Phase-1 metric name set is pinned NOW, not when each
   call site lands** (`core/pkg/wkmetrics/wkmetrics.go:16` namespace
   constant + `wkmetrics.go:171-256` collector declarations). Renaming a
   Prometheus metric or label is a Grafana-dashboard-breaking change;
   pinning the contract at the package-creation point means future
   instrumentation sub-items (LLM token spend, tool invocations,
   rate-limit headroom) plug into a stable name/label surface without
   re-litigating naming. The metric-family list is duplicated in
   `wkmetrics_test.go:expectedFamilies` and asserted by
   `TestNew_RegistersExpectedFamilies` — a drift catches loudly in CI.

3. **Typed-enum label values beat source-grep AC.** Iter-1 review M4
   asked for a source-grep that would assert no free-form outcome
   strings landed in `WithLabelValues(...)` call sites. The cleaner
   answer is `type Outcome string` + `OutcomeSuccess` / `OutcomeError`
   typed constants (`wkmetrics.go:23-34`). Recorder signatures accept
   the typed enum; a free-form `"ok"` is a compile error. The
   string aliases at line 39-43 stay so dynamic label composition
   (e.g. label values read from external config) still works.

4. **Vec metrics do not appear in `Registry().Gather()` until at least
   one label combination has been touched**
   (`core/pkg/wkmetrics/wkmetrics_test.go:54-79` `touchAllFamilies`
   helper). Tests asserting "the family is registered" MUST first
   touch a label combination per family; otherwise the assertion
   fails on a freshly-built Metrics. The helper centralises the
   touch list so adding a metric family is one append rather than a
   per-test edit.

5. **HTTP latency middleware uses the static route template AND wraps
   the auth middleware**, not just the handler
   (`core/internal/keep/server/server.go:80-104`
   `NewRouterWithMetrics`). Iter-1 reversed the wrap order:
   `instrument(route, authed(h))` not `authed(instrument(route, h))`.
   This ensures 401 responses still register in the histogram so
   operators see auth-failure volume on the same panel as the
   happy-path traffic. The static route template (e.g.
   `"/v1/manifests/{manifest_id}"`) is the `route` label, NOT the
   raw URL path — cardinality stays bounded regardless of path
   parameters.

6. **Backwards-compat preserved by additive constructors, not signature
   changes** (`core/internal/keep/server/server.go:55-58, 80-89`
   `NewRouter` → `NewRouterWithMetrics`, `New` → `NewWithMetrics`).
   The original `NewRouter(v, pool, reg, heartbeat)` and
   `New(cfg, pool, v, reg)` are now thin wrappers that pass `nil`
   metrics. This kept the M2.7.\* test surface (handlers_read_test.go,
   handlers_write_test.go, handlers_subscribe_test.go,
   server_test.go) at zero changes. The router imports
   `wkmetrics` directly — the direction is allowed
   (`server → wkmetrics`); `wkmetrics` MUST NOT learn about HTTP
   server internals.

7. **`/metrics` mounted OUTSIDE the auth wall + operator opt-out for
   process collectors** (`core/internal/keep/server/server.go:73-75`
   mux.Handle precedes the `if v != nil` auth block;
   `wkmetrics.go:66-79` `Options.DisableProcessCollectors`).
   Prometheus scrapers typically authenticate by network ACL or
   sidecar; forcing a JWT here would prevent the dashboard stack
   from working out of the box. The deployment trade-off is that
   default `ProcessCollector` + `GoCollector` leak process-level
   info; operators behind a hostile network set
   `Options.DisableProcessCollectors: true` AND/OR ACL the listener.
   Contract regression-pinned by
   `TestNewRouterWithMetrics_MountsMetricsOutsideAuthWall` and
   `TestIter1_NewWithOptions_DisableProcessCollectors`.

8. **Structured logging adopts `log/slog` (stdlib), zero new logging
   dependency** (`core/pkg/wklog/wklog.go`). The package is a thin
   wrapper providing: (a) JSON handler factory with `subsystem`
   baked in via `slog.Logger.With`; (b) correlation-id injection via
   a typed `context.Context` key + custom `correlationHandler` so the
   id appears as a TOP-LEVEL JSON field, not a nested group; (c)
   env-driven level config with `WK_LOG_LEVEL_<UPPERCASE_SUBSYSTEM>`
   override and `WK_LOG_LEVEL` fallback; (d) loud one-shot warning on
   unrecognised level strings so a typo surfaces at boot rather than
   silently downgrading. Tests inject deterministic `LevelLookup`
   functions instead of `os.Setenv` so parallel sub-tests do not
   race on the global env (`wklog_test.go:staticLookup`).
   **Limitation pinned in iter-1**: `WithGroup` nests correlation_id
   under the group due to slog group semantics — documented honestly
   in the package doc + `TestWithGroup_CorrelationIDNestsUnderGroup`
   regression.

9. **The outbox success counter is committed atomically with the
   transaction** (`core/internal/keep/publish/outbox.go:209-260`).
   Iter-1 review C2 / Codex-P1b: bumping a Prometheus counter
   inside a transaction that can still roll back is a silent data-
   corruption pattern. The fix is to collect successful row IDs
   locally during the loop and emit ONE `OutcomeSuccess` sample per
   row AFTER `tx.Commit` returns nil. On Commit failure or upstream
   tick error, emit ONE batch-level `OutcomeError` instead of
   per-row over-counting.
